ROOT CAUSE ANALYSIS:
====================
The issue is clear now:

1. Dashboard Strategy Tab (StrategyOrchestratorViz):
   - Calls trpc.seerMulti.getOrchestratorState() with NO parameters
   - Gets REAL agent data from the first orchestrator (TechnicalAnalyst, OrderFlowAnalyst, etc.)
   - Shows: 49% Fast Score, 53% Total Confidence, 15% Threshold

2. Strategy Page Agent Breakdown Tab:
   - When "All Symbols" is selected, it uses aggregatedOrchestratorData
   - This creates FAKE aggregated agents (Aggregate Bullish Signals, Aggregate Bearish Signals, Average Confidence)
   - These are calculated from symbolStates.lastRecommendation, NOT from real agent signals
   - Shows: 0% Fast Score, 0% Total Confidence, 60% Threshold

The problem: When "All Symbols" is selected on Strategy page, it doesn't use the real orchestrator data.
Instead, it creates synthetic aggregated data from symbol recommendations.

FIX NEEDED:
- When "All Symbols" is selected, Strategy page should call getOrchestratorState() without parameters
  (same as Dashboard does) to get the real first orchestrator's agent data
- OR aggregate the actual agent signals from all orchestrators
